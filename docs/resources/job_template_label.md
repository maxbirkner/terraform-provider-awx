# awx_job_template_label (Resource)

Creates a label and associates it with a job template.

AWX labels are lightweight named tags that can be attached to job templates for
filtering and organisation purposes. When you apply this resource, AWX will either
create a new label with the given `name` inside the specified `organization_id`, or
reuse an existing label if one already exists with the same name in that organisation.
Destroying the resource **disassociates** the label from the job template; the label
object itself is not deleted from AWX (it may be shared with other templates).

## Example Usage

```terraform
data "awx_organization" "default" {
  name = "Default"
}

resource "awx_project" "example" {
  name            = "example-ansible-main"
  organization_id = data.awx_organization.default.id
  scm_type        = "git"
  scm_url         = "git@github.com/josh-silvas/example-ansible.git"
  scm_branch      = "main"
}

resource "awx_inventory" "example" {
  name            = "Example Inventory"
  organization_id = data.awx_organization.default.id
}

resource "awx_job_template" "example" {
  name         = "baseconfig"
  job_type     = "run"
  inventory_id = awx_inventory.example.id
  project_id   = awx_project.example.id
  playbook     = "master-configure-system.yml"
}

resource "awx_job_template_label" "example" {
  job_template_id = awx_job_template.example.id
  name            = "production"
  organization_id = data.awx_organization.default.id
}
```

## Schema

### Required

- `job_template_id` (Number) The ID of the job template to associate the label with.
- `name` (String) The name of the label. AWX will find or create a label with this name inside the given organization.
- `organization_id` (Number) The ID of the organization that owns the label.

### Read-Only

- `id` (String) The AWX ID of the label object. This is set after the resource is created.

## Import

Import is supported using the label ID (visible in the Terraform state after `apply`).
Note: because the import only receives the label ID, `job_template_id` and
`organization_id` must be set in the configuration so Terraform can reconstruct the
full state on the subsequent `plan`/`apply`.

```shell
terraform import awx_job_template_label.example 42
```
